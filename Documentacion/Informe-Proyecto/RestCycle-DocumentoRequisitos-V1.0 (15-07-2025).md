
# Requisito Funcionales y No Funcionales

Requisitos para el proyecto Rest Cycle

Requisitos
REQUISITOS FUNCIONALES

RF.1	REGISTRO DE USUARIO:
Registro para usuarios nuevos tomando datos esenciales para la creación de la cuenta.

RF.2	TERMINOS Y CONDICIONES: 
Aceptar términos y condiciones para registrarse. 

RF.3	REGISTRO DE USUARIO: 
Ingreso de usuarios con cuentas por medio de credenciales.

RF.4	CAMBIO DE CONTRASEÑA: 
Cambio de contraseña en caso de olvidarse por medio de códigos enviados a correos o teléfono.

RF.5	MEMBRESIAS: 
Elección de membresía ya sea gratuita o de pago.

RF.6	SELECCIÓN DE MODO: 
Acceso a cualquiera de los dos modos ya sea para controlar y generar hábitos saludables o el control parental.

RF.7	CREACION DE CUENTAS: 
Creación de cuentas para menores de edad, estas quedan enlazadas de una vez.

RF.8	VINCULACION DE CUENTA: 
Vincular cuentas con una cuenta que gestiona y la otra que es gestionada.

RF.9	CONTRASEÑAS DE SEGURIDAD: 
Creación de contraseña para el usuario gestionado.

RF.10	HISTORIAL DE USO: 
Ver un informe de las estadísticas de uso de aplicaciones del usuario.

RF.11	TAREAS PENDIENTES:
Ver tareas pendientes generadas por el usuario que gestiona.
RF.12	GESTION DE TAREAS PENDIENTES:
Para el usuario que gestiona; agregar, modificar y/o eliminar tareas, eventos o compromisos pendientes para el usuario gestionado.

RF.13	TIEMPO ADICIONAL POR TAREAS COMPLETADAS: 
Para el usuario que gestiona; modificar la cantidad de tiempo pare el uso de aplicaciones que dará completar la tarea.

RF.14	NOTIFICACIONES DE ACTIVIDADES: 
El sistema notifica una información de la actividad del usuario enlazado, sobre el uso de tiempo excesivo en aplicaciones del dispositivo. 

RF.15	NOTIFICACION DE USO EXEDIDO: 
Notificación al usuario que gestiona y el gestionado, cuando el usuario gestionado excedió el tiempo de uso en alguna aplicación.

RF.16	NOTIFICACION DE ACTIVIDADES: 
El sistema notifica al usuario que gestiona y gestionado las tareas y/o actividades registradas por el usuario que gestiona. 

RF.17	NOTIFICACION DE RECOMENDACIONES AL PADRE: 
El sistema notifica al usuario que gestiona, sobre actividades que se recomienda que haga el usuario enlazado para mejorar sus hábitos.

RF.18	BLOQUEO GENERAL: 
Bloqueo de todas las aplicaciones seleccionadas por el usuario que gestiona instantáneamente.

RF.19	NOTIFICACION DE INSTALACION: 
El sistema notifica al usuario que gestiona, el intento de instalación de una aplicación por parte del usuario gestionado.

RF.20	PERMISOS DE INSTALACION: 
El usuario que gestiona puede permitir o lo contrario la instalación de una aplicación por parte del usuario gestionado. 

RF.21	VINCULACION DEL CALENDARIO: 
El usuario podrá vincular la app con un calendario, para gestionarle las tareas y actividades que tiene pendiente el usuario enlazado. 

RF.22	TIEMPO MAXIMO DIARIO POR APLICACIÓN: 
El usuario tendrá la opción de establecer el tiempo de uso máximo diario en las aplicaciones seleccionadas del usuario gestionado.

RF.23	BLOQUEO TOTAL POR HORARIO: 
El usuario que gestiona podrá elegir un horario, en el cual el dispositivo del usuario gestionado se bloquea totalmente.

RF.24	INFORME DE TIEMPO DE USO DEL DISPOSITIVO: 
El usuario que gestiona podrá ver un informe con datos del tiempo que el usuario gestionado a estado utilizando el dispositivo móvil. 

RF.25	HISTORIAL DE NAVEGACION: 
El usuario que gestiona podrá ver el historial en navegadores, del usuario gestionado. 

RF.26	LISTA NEGRA SITIOS WED: 
El usuario que gestiona podrá agregar a una lista negra de sitios web en los navegadores, esto bloqueará el ingreso a estos sitios del usuario gestionado.

RF.27	ACTIVACION GPS: 
El usuario gestionado activa el GPS con los permisos respectivos.

RF.28	REPORTE DE UBICACIÓN: 
El usuario que gestiona recibirá ubicaciones exactas donde estuvo el dispositivo del hijo.

RF.29	NOTIFICACION POR USO EXCESIVO: 
El sistema notifica cuando el usuario utilizó una aplicación después del tiempo límite. 

RF.30	NOTIFICACIONES MOTIVACIONES: 
El sistema notifica al usuario con mensajes motivacionales, que le ayudan a mejorar sus hábitos. 

RF.31	NOTIFICACIONES DE TEMA DE INTERES: 
El sistema notifica al usuario sobre temas de interés que haya seleccionado. 

RF.32	RECORDATORIOS DE ACTIVIDADES PENDIENTES: 
El sistema notifica al usuario de las actividades por realizar y con las fechas para realizarlas. 

RF.33	ALERTA USO PROLONGADO DE AURICULARES: 
El sistema notifica al usuario cuando haya usado audífonos por demasiado tiempo. 

RF.34	ENLACE CON OTROS APLICACIONES: 
Enlace a aplicaciones que se desean gestionar su tiempo de uso diario. 

RF.35	TIEMPO MAXIMO POR EXCESO DE TIEMPO: 
Establecer el tiempo máximo para el uso diario de una aplicación.

RF.36	MEDIDAS POR EXCESO DE TIEMPO: 
Seleccionar el tipo de medida que se toma al alcanzar el tiempo máximo de uso de una aplicación. 

RF.37	INFORME DE TIEMPO EN APLICACIONES: 
Solicitar un informe del tiempo que se utilizó cada aplicación y también filtrar por el tipo de aplicación. 

RF.38	HORARIO DE DESCANSO: 
Establecer horas de descanso donde el dispositivo no se usa.

RF.39	MEDIDAS DURANTE EL DESCANSO: 
Establecer medidas que se toman durante la hora de descanso.

RF.40	CONEXIÓN CON CALENDARIO: 
Enlazar la aplicación con el calendario del dispositivo.

RF.41	GESTION DE TAREAS: 
El usuario podrá agendar tareas y compromisos pendientes, con algunos detalles. 

RF.42	ALMACENAMINETO DE INFORMACION RELEVANTE: 
Crear notas con título y su contenido. 

RF.43	ELECCION DE TEMAS DE INTERES: 
Seleccionar los temas de interés para el usuario.

RF.44	MODIFICACION DE TEMAS DE INTERES:
Cambiar los temas de interés para el usuario.

RF.45	ELIMINACION DE CUENTAS FRAUDULENTAS: 
El administrador podrá eliminar cuentas que presenten fallos o intentos de fraude en autentificación de datos.

RF.46	SUGERENCIA PARA MEJORAR: 
El administrador podrá hacer sugerencias a los programadores para editar las interfaces o funciones.

RF.47	GESTION DE QUEJAS Y RECLAMOS: 
El administrador consultara y gestionar las quejas y reclamos de los usuarios.

RF.48	REPORTES PLANES PREMIUM: 
El administrador podrá ver el reporte de los planes premium que se encuentran activos.

RF.49	LISTADO DE CUENTAS:
El administrador podrá generar un listado de cuentas comunes y premium para su debida gestión.

RF.50	CUENTAS PREMIUM VENCIDAS: 
El administrador revisara y gestionar las cuentas premium vencidas.

RF.51	PAGOS Y CONTRATOS: 
El administrador gestionara los pagos y los contratos con las empresas publicitarias.

RF.52	ENVIO DE OFERTAS: 
El administrador podrá generar un listado para enviar ofertas a los usuarios registrados.

RF.53	CONTRATOS CON PATROCINADORES: 
El administrador gestionara los contratos con patrocinadores.

RF.54	CAMBIO DE CONTRASEÑA: 
El usuario puede modificar su contraseña.

RF.55	SELECCIÓN DE IDIOMA: 
El usuario puede cambiar el idioma de la aplicación.

RF.56	CONTACTO CON SOPORTE TECNICO: 
El usuario puede comunicarse con soporte técnico.

RF.57	EDICION DE INFORMACION PERSONAL: 
El usuario puede modificar su información personal.

RF.58	TEMA CLARO/ OSCURO: 
El usuario podrá cambiar entre tema oscuro y tema claro.

REQUISITOS NO FUNCIONALES

RNF.01		El sistema notifica a través de una burbuja tipo chat, esta notificación debe enviarse y generarse entre 5 a 7 segundos de respuesta.

RNF.02		La interfaz debe ser fácil de manejar con un diseño atractivo y muy accesible.

RNF.03		La aplicación debe ser accesible y que cuente con un soporte en varios idiomas.

RNF.04		La aplicación debe responder a las interacciones con el usuario en menos de 4 segundos.

RNF.05		La aplicación debe ser capaz de manejar mil usuarios simultáneamente sin degradar el rendimiento del sistema.
 RNF.06	El sistema debe estar disponible las 24 horas del día, los 7 días de la semana. 

RNF.07		La aplicación debe garantizar la protección de los datos obtenidos del usuario mediante un cifrado de datos y medidas de seguridad robustas.

RNF.08		La aplicación debe ser compatible con la mayoría de las versiones de sistemas operativos móviles y con los más recientes.

RNF.09		La aplicación debe funcionar correctamente en la mayoría de los dispositivos móviles.

RNF.10		Las actualizaciones de la aplicación no deben afectar la integridad de los datos existentes.

RNF.11		La aplicación debe contar con un proceso de control de calidad, para que las nuevas versiones no lleven errores.

RNF.12		La aplicación debe soportar un aumento del 50% de los usuarios concurrentes sin afectar integridad.
